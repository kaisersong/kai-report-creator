## 设计理念章节内容要点

每个子章节必须包含：

| 子章节 | 核心内容 |
|--------|----------|
| 1. 渐进式披露 | 命令路由表 + "规则在技能里，资产在文件里" |
| 2. 硅碳协作设计 | 输入端 IR（Frontmatter/正文/组件块）+ 输出端三层 AI 可读结构 + "渐进式披露为双物种设计" |
| 3. 视觉节奏 | 禁止连续 3 个纯文字章节 + 认知节拍需要 |
| 4. 异步决策支持 | 无讲述者在场 + 产品设计约束 + 判断标准 |
| 5. 设计质量基线 | 90/8/2 配色 + 10:1 字号张力 + KPI 网格规则 + 内容气质色调 |

## 功能分组定义

| 分类 | 包含功能 |
|------|----------|
| 核心功能 | 零依赖、主题、组件类型、Review 系统、AI 可读输出 |
| 交互功能 | 摘要卡片、内置导出、移动端自适应、双语 |
| 输出功能 | 自定义主题、自定义模板、主题覆盖、离线打包 |

## 双语一致要求

1. 章节顺序完全一致
2. 章节标题翻译一致
3. 表格内容翻译一致
4. 英文版用英文，中文版用中文，不混用

## 版本日志格式

每版本一行，格式：`**vX.Y.Z** — 简述（约 30 字）`

最新版本在最前，不展开详细 changelog。

## 禁止内容

- 版本日志不得放在开头
- 功能列表不得平铺（必须分组）
- 设计理念不得精简过度
